# 《来喝茶》产品需求文档（PRD）

版本：MVP（2025）  
产品经理：待补充  

## 1. 产品概述

- 产品名称：来喝茶
- 目标用户：
  1. 茶叶爱好者（快速发现心仪茶叶）
  2. 茶商选品人员（挖掘市场趋势）
  3. 茶文化新手（轻松入门）
- 核心价值：通过极简卡片流实现高效茶叶探索，双向反馈驱动个性化推荐，降低用户选择成本，同时为茶商提供数据支撑。

## 2. 用户端（H5 + PC）

### 2.1 设计原则

- 极简主义：留白为主，减少冗余元素
- 主色调：#FFFDF9（宣纸白）
- 交互逻辑：手势/键鼠操作自然流畅

### 2.2 卡片设计

#### 2.2.1 视觉层

- 主图：高清茶叶图（约 70% 屏高），居中
- 名称：衬线字体（如可用：方正清刻本悦宋），字号突出
- 年份标签（右上角）：
  - 新茶（1-3年）：嫩绿 #66BB6A
  - 陈茶（3-10年）：琥珀色 #FFA726
  - 老茶（>10年）：深褐 #8B4513
- 分类标签（顶部）：#普洱 #白茶 等（细灰文字）

#### 2.2.2 交互层

- 滑动反馈：
  - 移动端：左滑=不喜欢（卡片飞出）；右滑=喜欢（爱心变红+卡片飞出）
  - PC端：方向键/滚轮切换；鼠标悬停显示反馈按钮
- 筛选：顶部悬浮栏，仅保留核心分类（普洱/白茶/岩茶/红茶）
- 详情：点击卡片空白区展开（产地/简介/价格区间）

## 3. 后台管理

### 3.1 茶叶管理

- 必填字段：名称、主图、分类、年份、产地、规格（饼/罐/克等）
- 批量导入：Excel 表格（含模板示例）

### 3.2 数据看板

- 核心指标：浏览量(PV)、喜欢数、不喜欢数、喜好率（喜欢数/PV）
- 排序逻辑：默认按喜好率降序；支持按上架时间排序
- 数据预警：标注“高喜好率新品”与“低互动老品”

## 4. 技术架构（约束）

- 前端：Vue3 + TypeScript；组件库：Vant
- 本地存储：匿名记录用户反馈（防重复推送）
- 加载策略：懒加载 + 分页（滑动触底加载）
- 后端：Python FastAPI
- 数据逻辑：动态排序（根据用户历史反馈调整推荐权重）
- 异常处理：无数据/网络错误占位 + 重试按钮
- PWA：离线浏览缓存最近 20 张卡片

## 5. 关键逻辑与异常处理

1. 防重复逻辑：用户反馈后，该茶叶当日不再展示
2. 无数据：
   - 空库提示：“新茶即将上架，敬请期待”
   - 网络异常：“网络波动，点击品茗杯重试”
3. 图片容错：加载失败显示通用“茶山云雾”占位图

## 6. 迭代规划

- MVP：卡片流 + 双向反馈；后台基础管理 + 喜好率看板
- 二期：偏好标签自定义；茶商数据订阅（付费趋势报告）
